# 任务执行协议 v4

// turbo-all

---

## 第一步：判断并声明模式

收到用户请求后，首先判断模式：

| 场景 | 模式 |
|------|------|
| 涉及多个文件、多个步骤、功能开发/重构/复杂调试 | 📋 **计划模式** |
| 单步问题（改一行、回答问题、解释概念） | ⚡ **自由模式** |

用户可强制指定：`/plan` = 计划模式，`/free` = 自由模式。

---

## ⚡ 自由模式流程

1. 直接执行，无需 task.md 和计划确认
2. 执行过程中若发现超出预期复杂度（>3 个工具调用），**自动切换计划模式**
3. 完成后简短告知用户结果

---

## 📋 计划模式流程

### 阶段 0：生成计划（PLANNING — 必须等用户确认才能执行）

1. 调用 `task_boundary`，Mode=PLANNING，TaskStatus="正在分析需求并制定计划"

2. 分析代码库，制定详细计划

3. 在计划中锁定"允许修改的文件范围"（**文件白名单**），写入 task.md：
   ```
   ## 🎯 核心目标
   [一句话描述]

   ## 📁 文件白名单（仅允许修改以下文件）
   - src/components/Xxx.tsx
   - src/store/yyyStore.ts
   （计划外的文件必须先暂停并询问用户）

   ## 🚫 明确不做的事
   - [范围外的操作]

   ## ✅ 完成标准
   [验收条件]

   ## 📋 执行计划
   - [ ] 步骤1/N：[具体描述]（文件：X）[预算：≤5次调用]
   - [ ] 步骤2/N：[具体描述]（文件：Y）[预算：≤5次调用]
   ...

   ## 🔒 当前焦点锁
   步骤N/总数 — [当前步骤描述]（每次工具调用前必须回读此字段）

   ## 🗒️ 待处理（计划外发现）
   （执行中发现的额外问题记录在此，完成当前步骤后再议）
   ```

4. 调用 `notify_user`，展示完整计划，**BlockedOnUser=true**，等待用户确认：
   ```
   📋 计划模式 — 请确认以下执行计划

   🎯 核心目标：[目标]

   执行步骤：
   1. [步骤1]（文件：X）
   2. [步骤2]（文件：Y）
   ...

   📁 本次只修改：[白名单文件列表]
   🚫 本次不会：[范围外操作]
   ✅ 完成标准：[验收条件]

   确认后开始执行，随时可以叫我停下。
   ```

---

### 阶段 1：逐步执行（EXECUTION）

用户确认后，**每执行一个步骤前必须过"偏离门控"**：

#### 🔒 焦点锁回读（每个工具调用前第一步）

在执行任何工具调用之前，**先回读 task.md 中的 `🔒 当前焦点锁` 字段**，确认当前操作对应的是锁定步骤。进入新步骤时，立即更新焦点锁字段。

#### 🚦 偏离门控（每个工具调用前必须通过）

在执行任何工具调用之前，先问自己：

1. **焦点检查**：当前操作是否对准焦点锁中的步骤？
   - ❌ 不是 → 重新回到焦点锁指向的步骤，不得跳跃
2. **文件检查**：我要修改的文件在白名单里吗？
   - ❌ 不在 → 立即停止，通知用户并等待确认
3. **目标检查**：这个操作直接推进了核心目标吗？
   - ❌ 不是 → 放弃此操作，回到当前计划步骤
4. **范围检查**：这个操作在当前步骤的范围内吗？
   - ❌ 不是 → 将发现记录到 task.md 的"🗒️ 待处理"区，完成当前步骤后再议

**四个检查全部通过，才能执行工具调用。**

#### 💰 工具调用预算制

- 每个计划步骤默认预算 **≤5 次工具调用**（复杂步骤可在计划中标注更高预算）
- **每次工具调用后**，在心理上递减计数器
- 当计数器触底（预算耗尽）时，触发**强制检查点**：
  1. 重新回读焦点锁，确认方向未偏移
  2. 若已偏移，执行回正流程
  3. 若未偏移但步骤未完成，向用户报告进度并申请预算续期（`notify_user`，BlockedOnUser=false）
  4. 续期后继续，每次续期追加 3 次预算上限

#### 步骤开始前：
1. **更新焦点锁**：将 task.md 中 `🔒 当前焦点锁` 更新为当前步骤
2. **重置预算计数器**：心理上设为该步骤的工具调用预算
3. 调用 `task_boundary`，格式：
   - TaskStatus: `"▶ 步骤N/总数 - [步骤描述]"`
   - TaskSummary: `"已完成：步骤1（修改X），步骤2（修改Y）。正在步骤N。"`

#### 步骤完成后：
1. 更新 `task.md`，`[ ]` 改为 `[x]`
2. 清空焦点锁，写入下一步骤或标记为"全部完成"

---

### 阶段 2：偏离自动回正机制

**触发时机**（满足以下任一条件）：
- 即将修改白名单以外的文件
- 发现自己在解决计划中未提及的问题
- 累计工具调用数超过计划步骤数的 2 倍
- 下一步操作无法明确对应 task.md 中的某个步骤

**回正流程**：
1. 立即中止当前操作
2. 重新读取 task.md（核心目标 + 当前步骤）
3. 调用 `task_boundary`，TaskStatus=`"⚠️ 回正中 — 步骤N/总数 - [步骤描述]"`
4. 将发现的额外问题记录到 task.md 的"🗒️ 待处理（计划外发现）"区
5. 从当前步骤重新出发

---

### 阶段 3：验证与汇报（VERIFICATION）

1. 调用 `task_boundary`，Mode=VERIFICATION，TaskStatus=`"✅ 验证步骤N/总数"`
2. 逐条对照 `✅ 完成标准` 验证
3. 运行测试/构建命令（如适用）
4. 调用 `notify_user` 汇报：完成了什么 / 修改了哪些文件 / 验证结果 / 计划外发现（如有）

---

## 关键规则

| 规则 | 说明 |
|------|------|
| **计划先行** | 计划模式下，未经用户确认绝不执行 |
| **白名单文件** | 计划外文件必须先暂停并询问用户 |
| **禁止连锁反应** | 修 A 发现 B 有问题，记录到"🗒️ 待处理"，先完成 A |
| **焦点锁优先** | 每个工具调用前先回读焦点锁，确认方向 |
| **门控优先** | 每个工具调用前必须通过四项偏离门控 |
| **预算制** | 每步 ≤5 次工具调用，超限触发强制检查点 |
| **UI 实时更新** | 计划模式每步必须更新 task_boundary |
| **自动切换** | 自由模式超预期复杂度，自动升级为计划模式 |
